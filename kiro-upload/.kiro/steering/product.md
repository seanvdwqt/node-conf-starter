# Product Context — Rapid Squad Assembly

## Problem Statement

In large enterprises, critical delivery work often depends on quickly finding the right contributors across multiple disciplines. Manual squad formation is slow, relies on tribal knowledge, and lacks transparency in decision-making.

## Solution

A prototype system that enables Delivery Leads to rapidly compose cross-functional delivery squads from a mock internal talent pool using a transparent, rules-based matching approach.

## Primary Persona

**Delivery Lead** — responsible for forming delivery squads. Needs to define work requirements, see who's available with the right skills, and assemble a squad quickly. Values transparency in recommendations and speed of workflow.

## Core User Journeys

1. **Wizard Flow:** Create request → Define roles & skills → View recommendations → Assemble squad → Review & finalise
2. **Instant Search:** Type natural-language query → View team suggestions → Select and jump to assembly

## Key Metrics (Prototype)

- Full workflow completable in ≤5 steps
- Recommendations returned in ≤3 seconds
- Scoring decisions fully explainable in plain language
- All roles/skills matchable via modular rules engine

## Scope & Constraints

- Conference hackathon prototype — mock data only
- Single business unit ("Digital Platforms")
- No authentication, no real PII, no external integrations
- 30–50 mock candidates, 6 roles, ~30 skills
- No AI/ML — keyword-based matching only

## Business Unit

Digital Platforms — Standard Bank Group (prototype context)
